# C++基础补充

## 1、枚举（enum）

```c++
enum spectrum {red, orange, yellow, green, blue, violet, indigo, ultraviolet}
```

- 让 spectrum 成为新类型的名称；spectrum 被称为枚举（enumeration），就像 struct 变量被称为结构一样。
- 将 red、orange、yellow 等作为符号常量，它们对应整数值 0～7。这些常量叫作枚举量（enumerator）。

在默认情况下，将整数值赋给枚举量，第一个枚举量的值为 0，第二个枚举量的值为 1，依次类推。可以通过显式地指定整数值来覆盖默认值。

```c++
spectrum x;
x = blue;
x = 400;  //编译错误，枚举数据类型只能赋予定义时的值
```

实际上，枚举更常被用来定义相关的符号常量，而不是新类型。

## 2、使用new分配内存，delete删除

| 方面     | 传递地址（栈内存） | 使用new（堆内存） |
| -------- | ------------------ | ----------------- |
| 内存位置 | 栈（Stack）        | 堆（Heap）        |
| 分配方式 | 编译器自动管理     | 手动分配/释放     |
| 生命周期 | 作用域结束自动销毁 | 直到手动delete    |
| 性能     | 快                 | 相对较慢          |
| 大小限制 | 较小（栈空间有限） | 较大              |

现在都推荐使用智能指针

动态数组也推荐使用vector，当使用固定长度的数组时，建议使用array类

```c++
#include <array>
using namespace std;
array<typename, n> x; //n为元素数量，不能为变量
```

## 3、++i与i++

通常情况下没什么区别，但对于用户定义的类型，如果有用户定义的递增和递减运算符，则前缀格式的效率更高。

## 4、范围for循环

**范围for循环（range-based for loop）是C++11引入的语法糖，专为**简洁安全地遍历容器**而设计——它自动调用`begin()`/`end()`迭代器，**无需手动写索引或迭代器**，且**天然避免越界访问**。核心用途：**用`for (auto& x : container)`替代传统`for (int i=0; i<n; i++)`

| 类型        | 是否支持   | 示例                                 |
| ----------- | ---------- | ------------------------------------ |
| C++原生数组 | ✅          | `int arr = {1,2,3,4,5};`             |
| 标准库容器  | ✅          | `vector`、`map`、`string`、`array`等 |
| 自定义类型  | ⚠️ 条件支持 | 必须实现`begin()`和`end()`方法       |



| 声明写法                          | 适用场景                | 关键作用                 |
| --------------------------------- | ----------------------- | ------------------------ |
| `for (auto x : container)`        | 只读访问+元素是基本类型 | 拷贝元素（不修改原容器） |
| `for (auto& x : container)`       | 修改容器元素            | 引用修改原数据           |
| `for (const auto& x : container)` | 只读访问+元素是大对象   | 避免拷贝开销（推荐！）   |
| `for (auto&& x : container)`      | 通用（C++11起）         | 万能引用（完美转发场景） |

**数据结构必须有begin和end**
